01 · Opening
The Core Problem
We are paying more,
but owning less.
  • The same types of software are built again and again across different ministries
  • Systems do not communicate with each other — fragmentation grows with every new project
  • Many systems become inactive after a few years — a graveyard of unused tools
  • Most data is locked in PDFs — collected but never used for real decision-making

"We are paying multiple times for the same solutions — but not getting full value from them."

06 · Model
The Pilot Proposal
Start small. Prove value. Scale fast.
Phase 1 · Month 1–2
National Problem Registry
3–5 pilot ministries log real problems into an open platform visible to all contributors across Nepal
Phase 2 · Month 2–4
Open Data Dashboard
Extract usable data from PDFs. Simple dashboards for decision-makers. Public progress tracking
Phase 3 · Month 4–6
Full Demonstration
One complete cycle: problem published → solution contributed → expert review → government adoption

"We are not proposing a big change all at once — we are asking to start with one system and build a model Nepal can scale."
